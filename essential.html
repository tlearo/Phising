Descriptions

  • Only approved executables should run

    Prevent unapproved or malicious programs from executing by enforcing allow-lists.

  • Keep apps up to date

    Apply vendor patches quickly (e.g., browsers, PDF readers, Office) to reduce exploit windows.

  • Block risky macros

    Disable or restrict macros, especially from the Internet, and sign trusted ones only.

  • Harden user-facing apps

    Disable unnecessary features (e.g., Flash/Java, ads, dangerous MIME types) to reduce attack surface.

  • Least privilege

    Only grant admin rights when needed, use separate admin accounts, and monitor changes.

  • OS patching

    Apply security updates promptly across desktops, servers, and mobile devices.

  • Require MFA

    Add a second factor for remote access, privileged accounts, and high-value apps.

  • Back up and test restores

    Perform regular, tamper-resistant backups and verify you can restore quickly after incidents.
